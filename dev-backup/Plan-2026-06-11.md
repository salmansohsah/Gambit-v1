## 1. Project Overview

GAMBIT is a strategic digital agency website built on:
- **Framework:** Next.js 16.2.7 (App Router, Turbopack)
- **Styling:** Tailwind CSS v4
- **Database:** Supabase (PostgreSQL)
- **Hosting:** Vercel (auto-deploy from GitHub `main` branch)
- **Auth:** Supabase Auth (admin area protected)
- **CMS:** Partial — custom admin panel at `/admin` backed by Supabase

The site is **live on Vercel** and serves as the agency's primary digital presence. Visual branding and design language are complete and must be preserved. The primary gaps are Information Architecture, content completeness, Admin CMS coverage, and a robust synchronization/snapshot strategy.

## 3. UX Audit Summary

### Clarity Test: "30-Second Understanding"
A first-time visitor currently struggles to answer within 30 seconds:
- ✅ Who is GAMBIT? (Agency — communicated in hero badge)
- ❌ What does GAMBIT do? (Services buried in FrameworkSection)
- ❌ Why is GAMBIT different? (Philosophical paragraphs don't convert to clear differentiators)
- ✅ How does GAMBIT work? (ProcessSection handles this well)
- ✅ How to contact? (CTAs present)

**Score: 3/5 — Needs UX restructure**

### Identified UX Issues:
1. RealitySection — Two paragraphs of philosophy before any service explanation.
2. InitiativeSection — Abstract diagram and list. Doesn't help a visitor understand what to buy.
3. No explicit services listing with deliverables on the homepage.
4. Trust signals (client logos) are hardcoded placeholders.
5. Admin quick actions are not linked to their respective routes.

## 7. Synchronization Architecture

### Current Sync Flow
Admin edits → Supabase DB updated → Next.js cache invalidated via revalidateTag → Frontend reflects on next request. Already functional.

### Gaps:
1. No visual sync status in admin
2. No content snapshot/versioning
3. No Vercel deployment status visibility from admin

### Proposed Enhancements

#### Content Snapshot System
- New table: `content_snapshots` (id, created_at, created_by, label, snapshot_json JSONB)
- Admin triggers snapshot from Sync Center
- Snapshot captures: page_content, site_settings, navigation_items, seo_overrides
- Restore: writes snapshot data back + triggers revalidation

#### GitHub Integration (Read-only)
- GitHub API: GET /repos/{owner}/{repo}/commits?per_page=1
- Vercel API: GET /v6/deployments

#### Limitations
- Cannot push DB changes to GitHub (by design — content lives in Supabase)
- Cannot auto-sync local dev (requires same .env.local pointing to same Supabase project)
- Vercel API requires VERCEL_TOKEN env var

## 13. Rollback Strategy
1. Homepage IA: Revert page.tsx — 1 file, 1 git revert
2. New admin pages: Delete new route directories — no impact on existing
3. New DB tables: DROP TABLE for new tables — no existing data touched
4. Content changes: Use snapshot restore
5. Git: Atomic commits per phase — git revert per phase
